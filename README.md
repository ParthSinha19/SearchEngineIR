# Dense Passage Retrieval using Bi-Encoder + FAISS

A semantic document retrieval system built using a Transformer-based Bi-Encoder trained on the MS MARCO dataset and indexed using FAISS for efficient similarity search.

---

## Overview

Traditional lexical search methods such as TF-IDF and BM25 rely on exact term matching and fail to capture semantic similarity.  

This project implements **Dense Passage Retrieval (DPR-style architecture)** using contextual embeddings generated by BERT and performs efficient top-k similarity search using FAISS.

The system supports:

- Dense vector embedding generation
- Cosine similarity search
- Efficient indexing for large-scale corpora
- Top-k ranked passage retrieval

---

## Architecture

### Bi-Encoder Design

Two identical encoders (shared weights):


- Backbone: `bert-base-uncased`
- Embedding size: 768
- Similarity: Cosine similarity (via normalized dot product)
- Index type: FAISS `IndexFlatIP`

---

## Dataset

- Dataset: MS MARCO (v2.1)
- Passage-level retrieval task
- Subset used for indexing (configurable, e.g., 200k passages)

---

## Project Structure

Query → BERT → CLS Embedding → q
Passage → BERT → CLS Embedding → d
Similarity(q, d) = q · d

### How Retrieval Works
Query is tokenized
BERT generates CLS embedding
Embedding is normalized
FAISS performs nearest neighbor search
Top-k passages returned based on cosine similarity

### Training Objective

The model is trained using contrastive learning with in-batch negatives.
Loss function:

$L = -\log \frac{\exp(q \cdot d^+)}{\sum_{j=1}^{N} \exp(q \cdot d_j)}$

​
Where:
q = query embedding
𝑑+ = positive passage

Other passages in batch = negatives

### Evaluation Metrics

### Performance Considerations
Indexing 200k passages may take 1–3 hours on CPU
Search latency: milliseconds after index is built
Memory usage depends on corpus size
FAISS enables scalable approximate search

### Why Bi-Encoder?
Model Type	Accuracy	Speed	Scalability
Cross-Encoder	High	Slow	Low
Bi-Encoder	High	Fast	High
Bi-Encoder provides a strong trade-off between semantic quality and retrieval efficiency.
